# SystemDesign-GPT

SystemDesign-GPT is a production-ready MVP for an AI-powered system design interview assistant. It guides interviewers through clarifying questions, structured requirements, architectural reasoning, and Mermaid diagrams, then exports a Markdown-ready artifact.

## Features
- Streamlit single-page experience with loading states and session memory.
- Modular core with prompt templates, generator, schema validation, and diagram helpers.
- Clarification-first workflow: three questions before the final solution.
- Detailed sections for assumptions, requirements, architecture, database, scaling, caching, capacity math, bottlenecks, and tradeoffs.
- Mermaid architecture diagram rendered inline (falls back to self-generated graph if the LLM omits one).
- Markdown export button for sharing or record-keeping.

## Requirements
- Python 3.10+
- [OpenAI Python SDK](https://github.com/openai/openai-python)
- Streamlit UI toolkit
- Pydantic for structured output enforcement

## Setup
1. Copy the repo and create a virtual environment:
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Set your OpenAI key (never hardcode this value), either as an env var or in a `.env` file:
   ```bash
   export OPENAI_API_KEY="<your-key>"
   ```
   or
   ```bash
   echo 'OPENAI_API_KEY="<your-key>"' > .env
   ```

## Usage
1. Run the Streamlit app:
   ```bash
   streamlit run app.py
   ```
2. Run the FastAPI backend (optional API mode):
   ```bash
   uvicorn api:app --reload --port 8000
   ```
3. The Streamlit interface starts with an example question: **Design YouTube**. You can replace it with your own system.
4. Submit the question to prompt SystemDesign-GPT for three clarifying questions.
5. Provide answers to the clarifiers and generate the final solution with assumptions, diagrams, and tradeoff analysis.
6. Download the Markdown report via the **Export to Markdown** button.

## Notes
- Clarification questions and their answers remain in session state until a new interview prompt is submitted.
- The Mermaid diagram respects any valid flowchart generated by the LLM but also builds one from the component graph when missing.
- Error alerts surface if the OpenAI key is missing or the API returns malformed JSON.

## Sample prompt
> **Design YouTube**
>
> *Ask three clarifiers, then walk through assumptions, requirements, architecture, database, scaling, caching, capacity math, bottlenecks, tradeoffs, and surface a Mermaid diagram.*
