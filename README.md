# SystemDesign-GPT

This is a little Streamlit app I built to help practice for system design interviews. It uses an AI assistant (powered by OpenAI's `gpt-4o-mini`) to act as an interviewer. It asks clarifying questions first, then generates a full design document, including an architecture diagram.

## Features

- **A proper interview flow:** You give it a prompt(like "Design YouTube"), it asks three clarifying questions, and then it generates the full design based on your answers.
- **Detailed output:** The final design covers everything from requirements and architecture to database design, scaling, and bottlenecks.
- **Architecture diagrams:** It generates a Mermaid diagram to visualize the architecture. If the AI messes up the diagram, the app tries to build a fallback from the components list.
- **Markdown export:** You can download the whole thing to a Markdown file when you're done.
- **Simple and fast:** It's a single-page Streamlit app that uses `gpt-4o-mini` to keep things quick and cheap.

## Requirements
- Python 3.10+
- [OpenAI Python SDK](https://github.com/openai/openai-python)
- Streamlit UI toolkit
- Pydantic for structured output enforcement

## Setup
1. Copy the repo and create a virtual environment:
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Set your OpenAI key (never hardcode this value), either as an env var or in a `.env` file:
   ```bash
   export OPENAI_API_KEY="<your-key>"
   ```
   or
   ```bash
   echo 'OPENAI_API_KEY="<your-key>"' > .env
   ```

## Usage
1. Run the Streamlit app:
   ```bash
   streamlit run app.py
   ```
2. Run the FastAPI backend (optional API mode):
   ```bash
   uvicorn api:app --reload --port 8000
   ```
3. The Streamlit interface starts with an example question: **Design YouTube**. You can replace it with your own system.
4. Submit the question to prompt SystemDesign-GPT for three clarifying questions.
5. Provide answers to the clarifiers and generate the final solution with assumptions, diagrams, and tradeoff analysis.
6. Download the Markdown report via the **Export to Markdown** button.

## Notes
- Clarification questions and their answers remain in session state until a new interview prompt is submitted.
- The Mermaid diagram respects any valid flowchart generated by the LLM but also builds one from the component graph when missing.
- Error alerts surface if the OpenAI key is missing or the API returns malformed JSON.

## Example Workflow

**1. Input:**
> Design YouTube

**2. Clarification:**
The assistant asks 3 targeted questions to scope the problem (e.g., DAU, latency constraints, features).

**3. Output:**
A structured design document including assumptions, component breakdown, database schema, scaling strategies, and a visual architecture diagram.
